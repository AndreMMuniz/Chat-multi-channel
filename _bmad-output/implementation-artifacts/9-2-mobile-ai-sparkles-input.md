# Story 9.2: Mobile — AI Sparkles Button in Input

**Status:** done
**Epic:** 9 — Mobile Responsiveness
**Story Points:** 3
**Priority:** High
**Created:** 2026-05-02

---

## User Story

**As an agent on mobile,** I want a sparkles button inside the message input so that I can get AI-suggested replies with one tap, without the desktop suggestion panel taking up space.

---

## Background & Context

**Current desktop behavior:**
- An AI suggestion panel exists above the message input — shows suggested quick replies generated by the AI
- This panel is too large for mobile viewports and should not be visible on mobile

**Mobile target (defined in party mode roundtable 2026-05-02):**
- On `< md` (mobile): AI suggestion panel is `hidden`; a `TbSparkles` icon button appears inside the input row at the right side
- Tapping the sparkles button opens a Shadcn/UI `Sheet` from the bottom with the AI suggestions
- Selecting a suggestion inserts the text into the input and closes the Sheet
- On `md+` (desktop): sparkles button is `md:hidden`; the original inline suggestion panel remains

**Design system:**
- AI Accent color: `#7C3AED` — used EXCLUSIVELY for AI-generated content
- Sparkles icon: `TbSparkles` from `react-icons/tb`
- Idle state: `text-[#7C3AED] opacity-40`
- Active/loading state: `opacity-100` + spinner in same color
- Sheet: `side="bottom"` from Shadcn/UI

## Implementation Guide

### Sparkles Button Inside Input

```tsx
{/* Mobile-only sparkles button — inside the input row, right side */}
<button
  type="button"
  onClick={() => setAiSheetOpen(true)}
  disabled={isLoadingAI}
  className="md:hidden p-2 rounded-lg text-[#7C3AED] opacity-40 hover:opacity-100 disabled:opacity-20 transition-opacity flex items-center"
  aria-label="Get AI suggestion"
>
  {isLoadingAI
    ? <span className="w-4 h-4 border-2 border-[#7C3AED]/30 border-t-[#7C3AED] rounded-full animate-spin" />
    : <TbSparkles size={18} />
  }
</button>
```

Position: inside the input row `<div>`, after the text `<input>` and before the send button.

### Desktop AI Panel — Hidden on Mobile

```tsx
{/* AI suggestion panel — desktop only */}
<div className="hidden md:block">
  <AISuggestions ... />
</div>
```

### Mobile Sheet

```tsx
<Sheet open={aiSheetOpen} onOpenChange={setAiSheetOpen}>
  <SheetContent side="bottom" className="max-h-[50vh] overflow-y-auto pb-safe">
    <SheetHeader>
      <SheetTitle className="text-sm font-semibold text-[#7C3AED] flex items-center gap-2">
        <TbSparkles size={16} /> AI Suggestions
      </SheetTitle>
    </SheetHeader>
    <div className="mt-3 space-y-2">
      {suggestions.map((s) => (
        <button
          key={s.id}
          onClick={() => { onSelectSuggestion(s.text); setAiSheetOpen(false); }}
          className="w-full text-left px-4 py-3 rounded-xl bg-[#7C3AED]/5 border border-[#7C3AED]/20 text-sm text-slate-700 hover:bg-[#7C3AED]/10 transition-colors"
        >
          {s.text}
        </button>
      ))}
      {suggestions.length === 0 && (
        <p className="text-sm text-slate-400 text-center py-4">No suggestions available</p>
      )}
    </div>
  </SheetContent>
</Sheet>
```

Note: `pb-safe` should use `padding-bottom: env(safe-area-inset-bottom)` — add to Tailwind config or use inline style.

### Input Row Layout (mobile)

The input row needs to accommodate the sparkles button without overlapping the text. Use padding-right on the input:

```tsx
<input
  className="... pr-20 md:pr-4"  {/* extra right padding on mobile for the sparkles button */}
  ...
/>
```
